# Plano de Testes — ShopEasy

## Objetivo

Validar as principais funcionalidades do sistema de e-commerce ShopEasy, garantindo que os requisitos funcionais sejam atendidos e que os fluxos principais funcionem corretamente.

## Escopo

Serão testadas as seguintes funcionalidades:

- Cadastro de usuários
- Login
- Recuperação de senha
- Catálogo de produtos
- Carrinho de compras
- Aplicação de cupons
- Checkout
- Pagamento
- Controle de estoque

## Estratégia de Testes

Serão aplicados:

- Testes funcionais
- Testes exploratórios
- Testes de regressão
- Testes de sanidade
- Smoke tests

## Riscos

- Falta de dados de teste
- Requisitos incompletos ou alterados
- Ambiente de testes indisponível
- Falta de evidências visuais por se tratar de projeto simulado

## Critérios de Entrada

Os testes poderão ser iniciados quando:

- Os requisitos principais estiverem definidos
- As funcionalidades estiverem disponíveis para validação
- Os dados de teste estiverem preparados

## Critérios de Saída

Os testes serão considerados concluídos quando:

- Os casos de teste críticos forem executados
- Não existirem bugs críticos em aberto
- Os resultados forem documentados
- Os defeitos encontrados forem registrados

## Ferramentas Utilizadas

- GitHub
